# 语界 LinguaSpace 文档一致性检验报告

> 检验日期：2026-06-02
> 检验范围：`docs/文档/`、`frontend/`、`backend/`、`docker-compose.yml`、启动脚本

## 1. 总体结论

本轮已按当前仓库实现更新开发文档。项目当前是 React Web App + FastAPI 的本地 MVP：MySQL 保存运行时业务数据，Redis、MinIO、Neo4j 通过适配器接入，PostgreSQL/pgvector 已由 Docker Compose 启动但暂未承担正式检索；RAG 当前使用应用层哈希向量与余弦相似度检索。文本与视觉模型默认由 Ollama 提供，ASR 使用 faster-whisper，TTS 使用 Windows SAPI。

旧版文档将部分后续规划写成已落地实现，主要涉及 Vue、Milvus、PostgreSQL 主业务库、独立 Worker、Kubernetes 和监控套件。本轮已统一改为“当前实现”与“后续扩展方向”分离表达。

## 2. 当前实现基线

| 范围 | 当前实现 |
| --- | --- |
| 前端 | React 18、TypeScript、Vite、React Router |
| 后端 | FastAPI，单应用内按模块组织路由和业务编排 |
| 主业务库 | MySQL 8.4 |
| 知识检索 | `backend/app/embeddings.py` 的 256 维哈希向量与余弦相似度 |
| 基础设施 | Redis、MinIO、Neo4j 已接入；PostgreSQL/pgvector 为预留组件 |
| 模型服务 | Ollama 文本模型、Ollama Vision、faster-whisper、Windows SAPI |
| 部署 | Docker Compose 启动基础设施，PowerShell 脚本启动 FastAPI、Vite 和模型检查 |
| 角色入口 | 游客、学生、导游、知识库管理、系统管理均有 React 页面 |

## 3. 已修正文档差异

| 原文问题 | 修正结果 |
| --- | --- |
| 前端写为 Vue 3/H5/小程序 | 改为 React 18 + TypeScript + Vite Web App |
| PostgreSQL 被描述为主业务库 | 改为 MySQL 运行时主库 |
| Milvus 被描述为已接入向量库 | 改为应用层哈希向量检索，pgvector 作为后续增强 |
| Piper/MeloTTS 被描述为当前 TTS | 改为 Windows SAPI |
| vLLM、Transformers、NLLB、BGE 被描述为当前链路 | 改为 Ollama、术语表适配器和当前哈希向量实现 |
| Worker、Kubernetes、Prometheus/Grafana/Loki 被描述为当前部署 | 改为后续扩展方向 |
| 旧报告判断学生端缺失 | 已更正：当前存在 `/student/home`、`/student/scenarios`、`/student/training`、`/student/reports` |
| 旧报告判断 Neo4j 为内存降级 | 已更正：当前 `GraphMirrorAdapter` 直接连接 Neo4j，连接失败时启动失败 |
