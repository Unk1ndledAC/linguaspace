## 2. 文旅沉浸式提炼

### 可借鉴方向

- 首屏使用大幅景观图或本地生成图作为情绪入口，标题区保持高对比，CTA 数量克制。
- 顶部导航覆盖在 Hero 上方，首屏滚动后进入更清晰的内容分区。
- 卡片采用图文混排、较大留白和轻微悬浮反馈，避免后台式密集网格。
- 文旅端统一使用孔雀蓝、青绿色、米白和暖金色；图片上叠加柔和遮罩，保证文字可读。
- 场景、路线、文化提醒等内容使用旅行杂志式组织，不做普通管理表单。
- 动效控制在淡入、轻微上浮、图片小幅放大和处理链路逐步点亮。

### 适配到 LinguaSpace

| 页面 | 参考转化 |
| --- | --- |
| `/intro/overview` | 高冲击力 Hero + 玻璃浮层能力卡 + 真实知识统计 + 三个模块入口 |
| `/intro/architecture` | 文旅 Banner 下接半透明技术链路，不做 PPT 截图 |
| `/tourist/home` | 景区 Banner + H5 式快捷入口 + 真实内容推荐 |
| `/tourist/chat` | 文旅背景内嵌专业聊天工作区，右侧展示 RAG 来源和图谱增强 |
| `/tourist/voice` | 大号录音入口、链路时间线和讲解卡片 |
| `/tourist/image` | 移动拍照式上传区 + 识别摘要 + RAG 讲解来源 |
| `/guide/dashboard` | 文旅工作台，强调待接管会话、修正与知识回流 |

## 3. 简洁科技风提炼

### 可借鉴方向

- 左侧 Sidebar 稳定承载模块子导航，顶部状态栏展示面包屑、后端连接状态和用户信息。
- 页面标题、关键操作、筛选工具栏、数据区域形成固定阅读顺序。
- 卡片、表格、状态标签和按钮使用统一圆角、边框、间距和语义色。
- 深墨色与青绿色用于品牌关联，灰白背景用于信息密度控制，暖金色仅作少量强调。
- 图表区域保持克制，不用静态图片伪装趋势图或知识图谱。
- 空状态、加载状态、错误状态和接口待接入状态必须明显可区分。

### 适配到 LinguaSpace

| 页面 | 参考转化 |
| --- | --- |
| `/knowledge/documents` | 工具栏 + 上传区 + 文档/审核任务表格 |
| `/knowledge/graph` | 左侧筛选、中心真实图谱画布、右侧详情抽屉 |
| `/knowledge/rag-test` | 查询参数、检索结果、图谱关系、回答结果四块可追踪区域 |
| `/system/dashboard` | 健康状态、知识统计、日志摘要和接口待接入告警 |
| `/system/users` | 用户表格 + 新增表单 + 状态更新操作 |
| `/system/health` | 服务状态卡片 + 刷新操作 + 错误详情 |
| `/system/logs` | 筛选栏 + 模型调用日志 + 请求链路日志 |

## 4. 图谱交互提炼

- 图谱必须使用后端 `GET /api/graph` 返回的真实关系，不允许使用静态图代替。
- 节点由真实三元组中的 `source` 和 `target` 派生，边标签使用真实 `relation`。
- 画布提供关键词搜索、局部关系聚焦、刷新、关系新增和删除入口。
- 点击节点或边后打开详情区域；空图、请求失败和无搜索结果分别处理。
- 为保证展示稳定，布局可以使用前端确定性算法，但数据本身必须来自后端。

## 5. 双视觉体系

| 维度 | 文旅沉浸式 | 简洁科技风 |
| --- | --- | --- |
| 使用范围 | `/intro`、`/tourist`、`/guide` | `/knowledge`、`/system` |
| 布局 | 顶部导航、Hero、图文卡片、玻璃浮层 | Sidebar、状态栏、工具栏、表格、图谱 |
| 背景 | 本地图片、渐变、轻纹理 | 灰白或深墨面板、细网格、低饱和边框 |
| 色彩 | 孔雀蓝、青绿、米白、暖金 | 深墨、青绿、灰白、语义状态色 |
| 信息密度 | 适中，强调叙事与呼吸感 | 较高，强调扫描效率 |
| 动效 | 淡入、悬浮、轻微图片放大 | 状态刷新、Skeleton、局部高亮 |

## 6. 可借鉴与不可照搬

### 可借鉴

- 首屏构图、导航位置、留白节奏。
- 卡片比例、图文关系、工具栏组织。
- 图谱画布与详情联动方式。
- 后台信息层级、表格密度、状态标签语义。
- 克制的动效节奏。

### 不可照搬

- 参考站点品牌、Logo、原文案、原图片和远程资源。
- 参考站点完整页面排版。
- 与 LinguaSpace 业务无关的酒店、滑雪、商品或旅行产品字段。
- 静态截图冒充图表、图谱或真实系统数据。
- 任意联网下载的版权素材。

## 7. LinguaSpace 原创转化方案

1. 文旅端以“可信云南文化智能导览”为核心叙事，不复用参考站点文案。
2. 本地资产、CSS 渐变和 SVG 纹理优先；新增资源统一登记到 `docs/image-prompts.md`。
3. 文旅端和后台端共享青绿色品牌色，但布局、密度和图片策略明显区分。
4. 所有数值、图谱、日志、知识、会话和 AI 输出优先来自真实后端。
5. 后端缺失能力使用统一“接口待接入”状态，不以静态示例伪装成功。
