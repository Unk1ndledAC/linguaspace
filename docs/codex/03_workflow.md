# 03. Codex 工作流程

## 1. 总体流程

请按照以下流程推进：

1. 读取文档
2. 扫描后端接口和已有代码
3. 创建实施计划
4. 创建参考分析
5. 搭建工程骨架
6. 实现统一 API 层
7. 实现路由与 Layout
8. 分模块实现页面
9. 接入真实后端
10. 补充状态处理
11. 更新图片 Prompt 文档
12. 更新 README
13. 执行验证
14. 修复问题
15. 更新进度

## 2. 首先创建实施计划

创建或更新：

`docs/frontend-implementation-plan.md`

内容包括：

1. 已读取的设计文档清单
2. 已读取的参考网页清单
3. 完整路由树
4. 模块划分
5. 页面实现优先级
6. 每个页面对应的设计文档位置
7. 每个页面对应的参考网页类型
8. 每个页面需要调用的后端接口
9. 每个页面是否存在接口缺失
10. 每个页面需要的图片位
11. 每个阶段的完成状态
12. 当前阻塞问题

## 3. 创建参考分析文档

创建或更新：

`docs/reference-analysis.md`

内容包括：

1. 参考网页来源概览
2. 参考网页类型分类
3. 文旅沉浸式页面参考提炼
4. 简洁科技风页面参考提炼
5. 每个核心页面对应的参考来源
6. 可借鉴内容
7. 不可照搬内容
8. LinguaSpace 原创转化方案

## 4. 页面实现步骤

每实现一个页面，按以下步骤执行：

1. 阅读设计文档中该页面的章节。
2. 阅读参考文档中对应类型的参考说明。
3. 确认该页面需要的后端接口。
4. 在实施计划中记录接口映射。
5. 实现页面布局。
6. 接入真实接口。
7. 添加 loading 状态。
8. 添加 empty 状态。
9. 添加 error 状态。
10. 添加接口待接入状态。
11. 添加权限判断。
12. 更新图片 Prompt 文档。
13. 更新实施计划完成状态。
14. 运行检查。

## 5. 分阶段推进

### 第一阶段：工程基础

目标：

- 路由系统完成
- Layout 完成
- API Client 完成
- 导航完成
- 状态组件完成
- 文档计划完成

### 第二阶段：核心展示与游客功能

优先实现：

- 项目介绍总览页
- 游客首页
- 多语问答页
- 语音导览页
- 拍照识别页

### 第三阶段：知识库核心功能

优先实现：

- 文档管理
- 知识切片
- 文化知识图谱
- RAG 检索测试
- 审核工作台

### 第四阶段：导游端功能

优先实现：

- 导游工作台
- 游客会话列表
- 会话详情
- 人工接管
- AI 回答修正
- 案例库

### 第五阶段：系统管理功能

优先实现：

- 系统总览
- 用户管理
- 角色权限
- 接口健康监控
- 模型调用日志
- 指标分析
- 系统设置

### 第六阶段：统一打磨与验证

目标：

- 响应式优化
- 统一视觉细节
- 统一错误处理
- 统一空状态
- README 完善
- image-prompts.md 完善
- 构建检查通过
