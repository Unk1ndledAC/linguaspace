# 02. 上下文读取与限制

## 1. 必须读取的内容

开始开发前，必须读取：

1. `docs/LinguaSpace_frontend_page_design.md`
2. `docs/references.md`
3. 当前项目后端接口文档、OpenAPI、Swagger、路由、schema、DTO、README
4. 当前已有前端代码

## 2. 文档职责

### `docs/LinguaSpace_frontend_page_design.md`

决定：

- 完整路由结构
- 模块划分
- 每个子页面的功能
- 每个页面的布局方向
- 每个页面的视觉风格
- 每个页面的交互要求
- 每个页面的验收标准

### `docs/references.md`

决定：

- 不同类型页面可参考的网页或截图
- 可借鉴的首屏布局
- 可借鉴的卡片样式
- 可借鉴的导航方式
- 可借鉴的图片使用方式
- 可借鉴的视觉气质

### 后端接口文档 / OpenAPI / 路由代码

决定：

- 真实接口路径
- 请求方法
- 请求参数
- 响应字段
- 认证方式
- 上传方式
- SSE / WebSocket 是否可用

## 3. 冲突处理原则

如果页面设计文档、参考网页和真实后端接口发生冲突：

1. 真实后端接口优先。
2. 页面功能可根据真实接口做适配。
3. 缺失功能标注为“接口待接入”。
4. 在 `docs/frontend-implementation-plan.md` 中记录差异和处理方式。

## 4. 禁止事项

### 数据与接口

1. 不允许把 mock data 当正式数据源。
2. 不允许前端自己编造 AI 回答。
3. 不允许前端自己编造图谱数据当正式数据。
4. 不允许前端自己编造统计指标当正式数据。
5. 不允许点击按钮后只在前端假装成功。
6. 不允许杜撰后端接口字段。
7. 不允许在页面组件中散落 fetch / axios。
8. 不允许静默吞掉接口错误。

### 参考网页

可以借鉴：

- 页面结构
- 首屏构图
- 卡片样式
- 图文比例
- 导航方式
- 动效节奏
- 留白方式
- 色彩气质

不能复制：

- 品牌
- Logo
- 原文案
- 原图片
- 原网站完整排版
- 远程图片资源
- 有版权素材

### 图片资源

1. 不要联网下载图片。
2. 不要使用随机网络图片。
3. 不要使用版权图片。
4. 不要使用真实品牌 Logo。
5. 不要使用真实名人。
6. 前端中先用占位图、渐变、SVG 纹理或图片容器骨架。
7. 所有真实图片需求写入 `docs/image-prompts.md`。

### 视觉风格

1. 不要把文旅页面做成普通后台。
2. 不要把后台页面做成旅游官网。
3. 不要让所有页面使用同一种模板风格。
4. 不要为了视觉效果牺牲接口真实性。
5. 不要为了页面数量增加而做空页面。

## 5. Mock data 规则

允许：

1. 后端接口缺失时，页面显示“接口待接入”。
2. UI 骨架开发阶段使用 dev-only mock。
3. dev-only mock 必须放在明显目录，例如 `src/devOnlyMock/`。
4. 默认运行逻辑不能依赖 mock data。

禁止：

1. 使用 mock data 伪装真实后端数据。
2. 使用 mock data 让 CRUD 看起来已经真实可用。
3. 在 README 中把 mock 功能写成已接入功能。
