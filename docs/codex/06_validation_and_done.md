# 06. 验证、README 与完成标准

## 1. 每阶段验证

每完成一个阶段，执行可用验证命令。

优先执行：

```bash
npm run typecheck
npm run lint
npm run build
```

如果项目没有这些脚本，请查看 `package.json` 中已有脚本，并执行等价检查。

## 2. 验证失败处理

如果验证失败：

1. 优先修复明显错误。
2. 如果无法修复，记录失败原因。
3. 不要在失败未说明的情况下宣称完成。

## 3. 每个页面的最低验收标准

每个子页面至少应满足：

1. 页面有明确标题和功能定位。
2. 页面不是空壳。
3. 有真实接口接入或明确的接口待接入状态。
4. 有 loading 状态。
5. 有 error 状态。
6. 有 empty 状态。
7. 主要操作按钮有真实请求或明确禁用原因。
8. 布局符合所属视觉风格。
9. 页面内容和项目业务一致。
10. 不出现无关业务、无关字段或随机示例。

## 4. README 要求

创建或更新：

`README.md`

必须包含：

1. 项目启动方式
2. 环境变量配置
3. 后端接口地址配置
4. 技术栈说明
5. 完整路由表
6. 页面功能说明
7. 双视觉体系说明
8. 已接入接口列表
9. 缺失接口清单
10. Mock / 接口待接入说明
11. 图片资源使用说明
12. `docs/image-prompts.md` 使用说明
13. `docs/reference-analysis.md` 使用说明
14. 后续优化建议

README 必须与实际代码同步，不要写不存在的功能。

## 5. 进度记录要求

维护：

`docs/frontend-implementation-plan.md`

每完成一个模块或页面，都更新状态。

建议状态：

- 未开始
- 进行中
- 已完成
- 接口待接入
- 阻塞
- 需要复查

记录内容：

- 已完成页面
- 已接入接口
- 缺失接口
- 当前问题
- 下一步任务
- 验证结果

## 6. 最终完成标准

当且仅当满足以下条件，才认为目标完成：

1. 已读取并遵守 `docs/LinguaSpace_frontend_page_design.md`
2. 已读取并分析 `docs/references.md`
3. 已生成或更新 `docs/reference-analysis.md`
4. 已生成或更新 `docs/frontend-implementation-plan.md`
5. 多端多页面路由结构完成
6. 双布局体系完成
7. 项目介绍、游客端、导游端呈现文旅沉浸式风格
8. 知识库维护、系统管理呈现简洁科技风格
9. 页面优先接入真实后端接口
10. 缺失接口被明确标注
11. 所有页面具备 loading、error、empty 或接口待接入状态
12. 主要操作按钮有真实请求或明确禁用原因
13. `docs/image-prompts.md` 至少包含 35 条图片生成 Prompt
14. README 与实际实现一致
15. TypeScript / lint / build 检查通过，或明确记录无法通过的原因
